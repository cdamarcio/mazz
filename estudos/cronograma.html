Sistema de Gestão de Eventos

Cronograma Intensivo de 8 Semanas para Portfólio de Elite

React 18 + Vite Node + Fastify PostgreSQL + Prisma Docker

Semana 1: Fundação & Modelagem

Objetivo: Ter o banco e a API base rodando com Docker.

  • D1-2: Setup monorepo (/api, /web) e Docker Compose.
  • D3-4: Modelagem Prisma (Event, Ticket, Participant, User).
  • D5-7: Estrutura API (Controllers/Services) e Logger Pino.
Entrega: GET /health, migrations e seed de dados.

Semana 2: Autenticação & AuthZ

Objetivo: Sistema de login seguro e roles (Admin/User).

  • D1-2: JWT (Access/Refresh Tokens) e Bcrypt.
  • D3-4: Middleware de roles (Admin, Organizer, Attendee).
  • D5-7: Frontend: Context API e Proteção de Rotas.
Entrega: Fluxo completo de login com persistência segura.

Semana 3: CRUD de Eventos (Core)

Objetivo: Feature principal funcionando ponta a ponta.

  • D1-2: API: CRUD com Zod, paginação e filtros.
  • D3-4: React: Listagem e Cache com React Query.
  • D5-7: Forms com React Hook Form + Upload Imagem.
Entrega: Gerenciamento completo de eventos com interface polida.

Semana 4: Inscrições & Checkout

Objetivo: Lógica de negócio e transações de banco.

  • D1-2: DB Transactions (Atomicidade) para pagamentos.
  • D3-4: Controle de vagas com concorrência (SELECT FOR UPDATE).
  • D5-7: Webhook simulado e Dashboard de inscritos.
Entrega: Fluxo de inscrição robusto anti-furos.

Semana 5: Dashboard & Analytics

Objetivo: Dados agregados e visualização.

  • D1-2: Queries SQL complexas e EXPLAIN ANALYZE.
  • D3-4: Gráficos com Recharts e metrics cards.
  • D5-7: Exportação PDF e Performance (Code Splitting).
Entrega: Painel gerencial com performance otimizada.

Semana 6: Testes & Qualidade

Objetivo: Provar qualidade de código (QA).

  • D1-2: Testes unitários (Jest) em Services/Repositories.
  • D3-4: Integração (Supertest) nos fluxos críticos.
  • D5-7: Frontend tests (RTL + MSW) e CI no GitHub.
Entrega: Pipeline verde e cobertura > 70%.

Decisões Técnicas & Trade-offs

Situação Decisão Justificativa
ORM vs Raw SQL Prisma ORM Type-safety e velocidade de entrega.
Estado Global Context + React Query Evita complexidade do Redux para dados do servidor.
Arquitetura Monolito Modular Baixo overhead cognitivo para projeto solo.
